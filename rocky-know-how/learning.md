# 学习机制

## 什么触发学习

| 触发 | 置信度 | 行动 |
|------|--------|------|
| "不，应该 X" | 高 | 立即记录纠正 |
| "我之前说过..." | 高 | 标记为重复，提升优先级 |
| "总是/绝不 X" | 确认 | 晋升为偏好 |
| 用户编辑输出 | 中 | 记录为暂定模式 |
| 同纠正 3 次 | 确认 | 询问确认作为规则 |
| "在这个项目..." | 作用域 | 写入项目命名空间 |

## 什么不触发学习

- 沉默（非确认）
- 任何单一实例
- 假设性讨论
- 第三方偏好（"John 喜欢..."）
- 群聊模式（除非用户确认）
- 隐含偏好（绝不推断）

## 纠正分类

### 按类型

| 类型 | 示例 | 命名空间 |
|------|------|---------|
| 格式 | "用 bullet 不要用 prose" | global |
| 技术 | "用 SQLite 不要用 Postgres" | domain/code |
| 沟通 | "消息更短" | global |
| 项目特定 | "这个项目用 Tailwind" | projects/{name} |
| 领域特定 | "Mac 上用 launchctl" | domains/infra |

### 按作用域

```
Global: 适用所有地方
  └── Domain: 适用类别（code, infra, dev）
       └── Project: 适用特定上下文
            └── Temporary: 只适用本次会话
```

## 确认流程

3 次类似纠正后：
```
Agent: "我注意到你更偏好 X 而不是 Y（已纠正3次）。
        我应该总是这样做吗？
        - 是的，总是
        - 只在 [上下文] 中
        - 不，逐案处理"

User: "是的，总是"

Agent: → 移动到已确认偏好
       → 从纠正计数中移除
       → 未来使用时引用来源
```

## 模式演进

### 阶段

1. **暂定** — 单一纠正，观察重复
2. **涌现** — 2次纠正，可能是模式
3. **待确认** — 3次纠正，询问确认
4. **已确认** — 用户批准，除非反转否则永久
5. **归档** — 90天+未使用，保留但不活跃

### 反转

用户可以随时反转：
```
User: "其实，我对 X 改变主意了"

Agent:
1. 归档旧模式（保留历史）
2. 用时间戳记录反转
3. 将新偏好添加为暂定
4. "明白了。我现在用 Y。（之前：X，已归档）"
```

## 反模式

### 绝不学习

- 什么让用户更快服从（操纵）
- 情感触发或弱点
- 来自其他用户的模式（即使共享设备）
- 感觉"令人毛骨悚然"的内容

### 避免

- 从单一实例过度泛化
- 学风格而非实质
- 假设偏好稳定
- 忽略上下文变化

## 质量信号

### 好的学习

- 用户明确说明偏好
- 模式在上下文中一致
- 纠正改善结果
- 用户确认时询问

### 坏的学习

- 从沉默推断
- 矛盾最近行为
- 只在窄上下文有效
- 用户从未确认
